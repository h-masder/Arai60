## おまけ2　`O(1)`のメモリ空間で解く方法
他の方々のコメントを参考にFloydのアルゴリズムを使う方法を実装してみる。
全部は見ずに、自分に対して少しのヒントを与えて考えてみることにした。

与えたヒント：ポインタを二つ用意して、まず **slow**（1歩ずつ）と **fast**（2歩ずつ）を動かせばよい

これでCycleがあることは判定できる。なぜなら「どちらも一回動かすとその距離は1歩ずつ離れていく。したがって、Cycleになっていればかならず衝突する（出会う）」から。こちらは難なく実装できた。

一方で、先頭を出力する方法はまったく見当がつかなかった。
いろいろな人のコードをみて、以下のように書かれていた。
```
        slow = head
        while True:
            if slow is fast:
                return slow
            slow = slow.next
            fast = fast.next
```
このコードから「（スタート地点からCycle開始地点との距離）＝（衝突点からCycle開始地点との距離）」
が導かれていることを類推したが、
「slowとfastが衝突したとき、fastの進んだ距離は、slowの進んだ距離の2倍」ということから
「（スタート地点からCycle開始地点との距離）＝（衝突点からCycle開始地点との距離）」
を導けなくて苦労した。結局あれこれ考えて納得し、その後、以下のコメントをみて別の方法のあるのだなと納得した。手順自体は問題なく実装できた。
(https://discord.com/channels/1084280443945353267/1246383603122966570/1252209488815984710)
code3.pyが書いたコード。

一夜たち、あれこれ考えたほうはもうどうやって解いたのか覚えていなかった。コメントのほうは覚えていた。結局、コメントの考え方が直感でわかりやすいのだと思った。

今回のコーディングで思ったことは二つ。
- 誰かに教えるとき（今回は自分自身）、その人の思考のクセや背景知識を考慮し、その人が理解できるようなコメントを残すのは本当に難しい。これは、たくさん会話をしてつかむしかないのだろうか。その練習として、当面は同じ人にレビューを頼んでみようと思った。
- 以下のコードをみてCycleの開始地点が求められていることを自分はすぐには理解できない。
```
        slow = head
        while True:
            if slow is fast:
                return slow
            slow = slow.next
            fast = fast.next
```
